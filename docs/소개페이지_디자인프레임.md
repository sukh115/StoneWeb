# 돌과 시간 — 게임 소개 페이지 디자인 프레임 v1
> 2026-07-06 · 작성: Claude (StoryWiki 기반) · 목적: 아트가 확보되는 대로 이 설계도대로 조립
> 현행 티저(돌과시간_티저.html)는 이 프레임의 축약판 — 섹션별로 점진 교체 가능

## 0. 설계 원칙

1. **페이지 = 게임의 재연.** 스크롤이 곧 여정 (1→10→색). 맨 위는 완전 흑백, 내려갈수록 어두워졌다가(중심지), 마지막에 색이 터진다. 게임의 명암 설계("가장자리 옅은 회색 → 중심은 순백과 칠흑")를 페이지 수직축에 그대로 매핑
2. **대사 0 톤 유지.** 카피는 섹션당 최대 2문장. 설명하지 말고 보여주기. 꿈보다 해몽 — 스포일러(격변의 정체, 진엔딩) 절대 노출 금지
3. **인터랙션은 게임 메카닉의 은유로만.** 장식용 이펙트 금지 — 모든 인터랙션이 사륜안/강도/색 중 하나를 체험시켜야 함
4. 모바일 우선 반응형, 이미지 lazy-load, 단일 HTML 유지 (배포 단순화)

## 1. 전체 구조 — 스크롤 시나리오 (9섹션)

```
[S0 인트로] → [S1 히어로] → [S2 세계] → [S3 골렘] → [S4 시스템 3종]
→ [S5 재질 체험] → [S6 여정 파노라마] → [S7 색의 귀환] → [S8 사운드] → [S9 CTA]
   흑백 ──────────────────────── 점점 어두움 ──────────┘ 컬러 ─────────→
```

### S0 · 인트로 (첫 3초)
```
┌─────────────────────────┐
│                         │   완전한 검정.
│                         │   중앙에 조약돌 하나가 또르르 구른다 (CSS 애니 1회)
│          ●  ←돌         │   → 멈춤 → 눈에 흰 빛이 켜짐 → 페이드로 S1
│                         │   (프롤로그 "첫 움직임" 오마주, 스킵 가능)
└─────────────────────────┘
```
- 필요 에셋: 조약돌 1개 스프라이트 (or CSS 도형으로 대체 가능)
- 소리: 무음 → 돌 구르는 소리 1회 (유저 인터랙션 후에만, 브라우저 정책)

### S1 · 히어로
```
┌─────────────────────────┐
│ 원경: 안개 숲 (패럴랙스 느림) │  구도: 3레이어 패럴랙스.
│   중경: 플랫폼·사슬 (중간)    │  골렘은 화면 좌하 1/3 지점 (여백이 세계의 적막을 말함)
│  [골렘]                  │  타이틀은 우측 상단이 아닌 **하단 중앙** —
│      돌과 시간            │  시선 흐름: 골렘(좌하) → 타이틀(중하) → 스크롤 유도(하단)
│      STONE & TIME  ↓     │
└─────────────────────────┘
```
- 필요 에셋: 숲 배경 3레이어 분리 PNG (원경 실루엣 / 중경 플랫폼 / 전경 풀·바위), 16:9 이상 와이드. 골렘 idle 애니 (스프라이트시트 6~8프레임)
- 인터랙션: 마우스 이동에 미세 패럴랙스. 골렘 idle은 "정지 자세뿐" — 1막 초기 상태 그대로 (내면 채널 A의 0단계를 여기서부터 재연)

### S2 · 세계 — 색을 빼앗긴 숲
```
┌─────────────────────────┐
│ 텍스트 40%  │  아트 60%   │  좌우 분할. 데스크톱: 텍스트 좌·아트 우
│ "아주 오래전 │ [목각인형/  │  모바일: 상하 스택
│  누군가…"   │  폐허 아트]  │  아트는 스크롤 시 살짝 확대(1.0→1.05) — 들여다보는 느낌
└─────────────────────────┘
```
- 카피 2문장 고정 (현행 유지). 인류 정체는 절대 설명 안 함
- 필요 에셋: 목각인형 컨셉(보유) + 화석화된 인류 흔적 1컷 (돌 마을 or 폐허)

### S3 · 골렘 — 파츠 분해 인터랙션 ★
```
┌─────────────────────────┐
│        [머리]            │  중앙 대형 골렘. 스크롤 진입 시
│    [손]  [몸]  [손]       │  붙어 있던 파츠가 **살짝 벌어졌다 재조립**
│        [발] [발]          │  hover(터치) 시 해당 파츠만 떠오름
│  "서로 붙어 있지 않은      │  → Rayman식 부유형 구조 + 파츠 교체(장비)
│   돌들이, 서로를 찾는다"    │     시스템을 말 없이 전달
└─────────────────────────┘
```
- 필요 에셋: 골렘 파츠별 분리 PNG (머리/몸/좌우손/좌우발/낙엽 망토 — 게임용 PSD 파츠 재활용 가능, 추가 제작 0)
- 이게 이 페이지의 시그니처 인터랙션. 게임의 캐릭터 아키텍처가 곧 웹 연출이 됨

### S4 · 핵심 시스템 3종 — 스티키 스크롤
```
┌─────────────────────────┐
│ (고정 화면, 스크롤로 전환)  │  화면 고정(sticky) 상태에서 스크롤하면
│  ① 사륜안 ──────────     │  ①→②→③ 순서로 컷 전환.
│  [게임플레이 GIF 슬롯]     │  ① 사륜안: GIF가 재생 중 슬로모로 변속 +
│  "생각이 빨라지면,         │     배경 흐려짐 (로우패스의 시각 번역)
│   세상이 느려진다"         │  ② 강도: 타격 실패 크랙 연출 GIF
└─────────────────────────┘  ③ 색: 국소 회복 전후 비교 슬라이더
```
- 필요 에셋: 게임플레이 캡처 GIF/webm 3개 (데모 빌드에서 추출 — 데모 완성 후 삽입, 그전엔 정지 컷)
- ③은 이미지 비교 슬라이더(before 흑백/after 20% 채도) — 국소 회복을 직접 드래그로 체험

### S5 · 재질 체험 — 인터랙티브 ★
```
┌─────────────────────────┐
│       [골렘 대형]          │  버튼: 🪨화강암 ❄️얼음 🖤흑요석
│  🪨  ❄️  🖤              │  클릭 시 골렘 손·발 파츠의 질감 스왑 +
│ "부딪혀 보기 전엔 모른다"   │  바닥에 떨어지는 애니 속도 변화(무게) +
└─────────────────────────┘  타격음 1회 재생 (재질 사운드 어필)
```
- 필요 에셋: 파츠 3재질 버전 PNG (S3 파츠와 공용), 재질별 타격 SFX 3개 (사운드 문서 SFX 1주차 산출물 재활용)
- 게임의 "장비 슬롯 + 재질 물성 + 재질 사운드"를 30초 웹 체험으로 압축

### S6 · 여정 — 가로 파노라마
```
┌─────────────────────────┐
│ ←── 가로 스크롤 (세로 입력 변환) ──→
│ 숲기슭|높은숲|암벽|서리|기생|광산|화산|폐허|심근|중심지
│  옅은 회색 ────────────→ 순백과 칠흑
└─────────────────────────┘
```
- 세로 스크롤을 가로 이동으로 변환(sticky) — "숲을 가로지르는 깊이" 그대로
- 각 구간: 실루엣 아트 + 이름만. 보스는 실루엣 힌트 (암벽 골렘 정도만, 나머지 침묵)
- 필요 에셋: 구간별 실루엣 와이드 컷 10장 (팔레트 변주라 배치 부담 낮음 — 비주얼 패밀리 4~5개 재활용). **아트 부족 시**: 대표 4구간만 (데모 스코프와 일치: 숲기슭·암벽·심근·중심지)
- 진행 바가 명암 그라디언트로 채워짐 (제작 노트의 명암 대비 원칙)

### S7 · 색의 귀환 (피날레) ★★
```
┌─────────────────────────┐
│   (S6 끝 = 가장 어두움)    │  중심지 도달 직후, 스크롤 한 번에:
│      · 빛 한 줄기          │  화면 중앙에서 **원형(radial)으로 색이
│    ((( 색의 파도 )))       │  파도처럼 퍼짐** — clip-path circle 확장
│   흑백 → ■■컬러■■        │  + 동시에 BGM 칩튠→오케스트라 크로스페이드
└─────────────────────────┘  (사운드 ON 유저 한정)
```
- 현행 티저의 세로 그라데이션 전환을 **중심→바깥 원형 확산**으로 업그레이드 — 격변 연출("중심에서 바깥으로, 파도처럼") 정확 재연
- 필요 에셋: 같은 장면의 흑백/컬러 페어 1세트 (가장 중요한 발주 1순위. 지금은 꽃밭 이미지로 대체 중)
- 여기서 진화 스팅(15~40초 큐) 사용 — 사운드 문서 미제작 항목과 발주 연동

### S8 · 사운드 — 진화하는 음악
```
┌─────────────────────────┐
│ ♪ Ruins_Through_Pines    │  플레이어 1개 + [흑백|컬러] 토글 스위치
│ [▶] ──o────── [흑백|컬러] │  토글 시 같은 곡의 단조/장조 버전 크로스페이드
└─────────────────────────┘  = "진화하는 음악"을 직접 체험
```
- 필요 에셋: 채택곡 페어 1~2쌍 (로비1/2 등, 30초 발췌 mp3)
- S7 직후 배치라 "방금 들은 그 변화"를 손으로 재생해보는 구조

## 2. 비주얼 시스템

| 항목 | 값 |
|---|---|
| 배경 | #0b0b0d (S0~S6) → 컬러 구간은 아트 주도 |
| 액센트 | #6ee7a0 (생명력 초록) — 컬러 도달 전엔 눈빛·진행바에만 절제 사용 |
| 타이포 | 제목: Pretendard ExtraBold, 자간 -2% / 본문 행간 2.0 / 영문 레터스페이싱 .3~.5em |
| 모션 | 등장: opacity+translateY 0.9s / 패럴랙스 최대 ±20px / 60fps 유지 위해 transform·opacity만 사용 |
| 사운드 | 기본 무음. 헤더에 🔊 토글 — 켜면 S7 크로스페이드·S5 타격음 활성 |

## 3. 아트 발주 우선순위 (이 페이지 기준)

1. **흑백/컬러 페어 장면 1세트** (S7 피날레) — 페이지의 존재 이유
2. 골렘 파츠 분리 PNG + 재질 3버전 (S3·S5) — 게임 PSD에서 추출, 신규 제작 최소
3. 히어로 배경 3레이어 (S1) — 보유 숲 컨셉아트 레이어 분리로 시작 가능
4. 구간 실루엣 4컷 (S6 축약판)
5. 게임플레이 GIF 3개 (S4) — 데모 빌드 이후
